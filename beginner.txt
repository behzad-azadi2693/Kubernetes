============minikube=============
install minikube for local machine

minikube is a one node cluster, where the master and worker processes are on the same machine

--->command
➜ minikube start --vm-driver=docker   
➜ minikube dashboard
➜ minikube stop
➜ minikube logs --file=logs.txt

============kubectl=============
install kubectl

kubectl, the command line tool for kubernets
you can run pods from three ways 1-yaml file 2-json file(for api) 3-cmmand kubectl run ...


--->command
➜ kubectl version --client                                               
➜ kubectl cluster-info
➜ kubectl convert --help               
➜ kubectl get node
➜ kubectl get pods
➜ kubectl describe pods/<pod name>
➜ kubectl describe nodes <node name>
➜ kubectl get services
➜ kubectl describe pod <name pod>
➜ kuebctl get rc
➜ kubectl run <name> --image=<image name> --port:<port> --generator=run/v1
➜ kubectl get po <pod name> -o yaml
➜ kubectl explain pod
➜ kubectl logs <pod name>
➜ kubectl port-forward <pod name> port:port
➜ kubectl create -f <file name>.yml
➜ kubectl describe pod <pod name>
➜ kubectl delete po --all
➜ kubectl delete <pod name>
➜ kubectl delete all --all

============pod=============
a pod is a group of one or more tightly related containers that will always run together on the same worker node and in the same linux nameespace, each pod is like a separate logical machine with its own ip, hostname, process and so on, running a single application
